Questão 1

Texto da questão

1)     Considerando os Princípios da Disponibilização. Assinale a alternativa incorreta.

Questão 1 Resposta
a.

     Software com bugs deve ser primeiramente corrigido e, depois, entregue, sob pressão do prazo, muitas empresas entregam incrementos de baixa qualidade, notificando o cliente de que os bugs “serão corrigidos na próxima versão”. Isso é errado.

b.

Um pacote de entrega completo deve ser montado e testado, todo software executável, arquivo de dados de suporte, documento de suporte e outras informações relevantes devem ser completamente montados e testados com usuários não necessariamente reais

c.

     As expectativas do cliente para o software devem ser gerenciadas, muitas vezes, o cliente espera mais do que a equipe havia prometido entregar e ocorre a decepção.

d.

     É preciso estabelecer uma estrutura de suporte antes da entrega do software, um usuário conta com o recebimento de informações precisas e com responsabilidade caso surja um problema. Se o suporte for improvisado ou, pior ainda, inexistente, imediatamente o cliente ficará insatisfeito.

e.

     Material instrucional adequado deve ser fornecido aos usuários, a equipe de software deve entregar mais do que o software em si. Auxílio em treinamento de forma adequada e orientações quanto a problemas inesperados.

Texto da questão

1)     Considerando os Princípios da Modelagem de Requisitos. Assinale a alternativa incorreta.

Questão 2 Resposta
a.

     O domínio de informações engloba os dados constantes no sistema, os dados que fluem para fora do sistema e a armazenagem de dados que coleta e organiza objetos de dados devem ser mantidos permanentemente.

b.

     A modelagem de análise se inicia pela descrição do problema sob o ponto de vista do usuário. A essência do problema é descrita sem levar em consideração como será implementada uma solução.

c.

As funções oferecem benefício direto aos usuários. Algumas transformam dados que fluem no sistema, outras exercem um certo nível de controle sobre o processamento

d.

     O comportamento de um software é comandado por sua interação com o ambiente externo. Dados fornecidos pelos usuários ou dados de monitoramento coletados de uma rede fazem o software comportar de maneira específica.

e.

     A modelagem de requisitos é a primeira etapa da solução de um problema de engenharia de software. Permite que se entenda melhor o problema e se estabeleçam bases para a solução.

f.

As funções do software oferecem benefício indireto aos usuários e suporte interno para fatores visíveis aos usuários. Algumas funções transformam dados que fluem no sistema, outras exercem certo nível de controle sobre o processamento interno do software

Texto da questão

1)     Considerando os Princípios de Construção. Assinale a alternativa incorreta.

Questão 4 Resposta
a.

      Todos os testes devem estar alinhados com os requisitos do cliente

b.

      Nos princípios da codificação, restrinja seus algoritmos seguindo a prática de programação estruturada, pense na possibilidade de usar programação em pares, selecione estruturas de dados que atendam às necessidades do projeto, domine a arquitetura de software e crie interfaces coerentes com ela.

c.

     Os princípios que regem a codificação são intimamente alinhados ao estilo às linguagens e aos métodos de programação.

d.

      Testes exaustivos são impossíveis.

e.

     A atividade de construção abrange um conjunto de tarefas de codificação e teste que geram um software operacional pronto para ser entregue ao cliente ou usuário.

f.

     Nos princípios da validação, aplique uma revisão de código quando for apropriado, realize testes de unidades e corrija erros ainda não identificados, refatore o código para melhorar a sua qualidade.

g.

     Os testes devem ser planejados muito antes de serem iniciados.

h.

Nos princípios de testes, esses demonstram que as funções do software estão funcionando de acordo com as especificações e que os requisitos relativos ao desempenho e ao comportamento parecem não estar sendo atingido.

Texto da questão

1)     Considerando as Diretrizes da Qualidade de Software. Assinale a alternativa incorreta.

Questão 5 Resposta
a.

     Um projeto deve exibir uma arquitetura que foi criada usando estilos ou padrões de arquitetura reconhecíveis.

b.

     Um projeto deve ser obtido usando-se um método repetível, isto é, dirigido por informações obtidas durante a análise de requisitos.

c.

     Um projeto deve ser modular, ou seja, o software deve ser dividido logicamente em elementos ou subsistemas.

d.

Um projeto deve levar a interfaces que aumentam a complexidade das conexões entre os componentes e o ambiente externo.

e.

     Um projeto deve levar a estruturas de dados adequadas às classes a serem implementadas.

f.

     Um projeto deve conter representações distintas de dados, arquitetura, interfaces e componentes.

Texto da questão

1)     Considerando Engenharia de Requisitos. Assinale a alternativa incorreta.

Questão 6 Resposta
a.

     Deve-se solicitar a clientes, usuários e outros envolvidos para que ordenem seus requisitos e discutam sua prioridade. Em uma negociação efetiva não existem ganhadores nem perdedores.

b.

     Na concepção do projeto, estabelecemos um entendimento básico do problema, das pessoas que querem uma solução e da natureza da solução desejada.

c.

     Entender as metas de negócios é muito importante no levantamento. Uma meta é um objetivo de longo prazo que um sistema ou produto deve alcançar.

d.

Entender os requisitos de um problema está entre as tarefas mais difíceis enfrentadas por um engenheiro de software

e.

     A especificação pode ser um documento por escrito, um conjunto de modelos gráficos, um modelo matemático formal, um conjunto de cenários de uso, um protótipo ou qualquer combinação dos fatores citados.

f.

     Muitos desenvolvedores querem começar imediatamente, antes de terem um entendimento claro daquilo que é necessário, a engenharia de requisitos estabelece uma base sólida para o projeto e para a construção.

g.

     A tarefa de elaboração concentra-se no desenvolvimento de um modelo de requisitos refinado que identifique os diversos aspectos da função, do comportamento e das informações do software. A elaboração é guiada pela criação e pelo refinamento de cenários dos usuários obtidos durante o levantamento.

h.

Uma preocupação fundamental no decorrer da validação de requisitos é a consistência. A validação de requisitos examina a especificação para garantir que todos os requisitos de software tenham sido declarados de forma ambígua.
